## 核心机制：二进制指针 + 最高位判断空满

### 指针结构

| 参数    | 值            | 说明                      |
|---------|--------------|---------------------------|
| `DEPTH` | 16           | FIFO 深度（必须是 2 的幂）  |
| `PTR_W` | 4            | 地址位宽，`$clog2(16) = 4` |
| 实际指针 | 5 位 `[4:0]` | 多 1 位用于空满判断         |

```
wr_ptr / rd_ptr 结构：
  [4]    [3:0]
   │       │
   │       └─► 实际地址（0-15）
   └─────────► 绕回标志位（MSB）
```

### 空满判断原理

| 条件     | 公式                                                        | 含义                 |
|----------|------------------------------------------------------------|----------------------|
| **空**   | `wr_ptr == rd_ptr`                                         | 读写指针完全相同       |
| **满**   | `wr_ptr[4] != rd_ptr[4]` 且 `wr_ptr[3:0] == rd_ptr[3:0]`   | 地址相同，但绕回位不同 |

**图示**：
```
初始：wr_ptr=00000, rd_ptr=00000  → 空（相等）

写满 16 个：
  wr_ptr = 10000 (绕回位=1, 地址=0000)
  rd_ptr = 00000 (绕回位=0, 地址=0000)
  → 满（地址相同，绕回位不同）

读 16 个：
  rd_ptr = 10000 (追上 wr_ptr)
  → 空（相等）
```

## 代码详解

### 1. 存储器阵列

```verilog
reg [DATA_W-1:0] mem [0:DEPTH-1];   // 数据存储
reg [PTR_W:0]    wr_ptr;             // 写指针（5位）
reg [PTR_W:0]    rd_ptr;             // 读指针（5位）
```

### 2. 写操作（时序逻辑）

```verilog
always @(posedge clk) begin
    if (wr_en && !full) begin         // 使能且不满
        mem[wr_addr] <= wdata;        // 写入数据
    end
end
```

**特点**：只检查 `!full`，满时自动忽略写入。

### 3. 读操作（时序逻辑，带复位）

```verilog
always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
        rdata <= {DATA_W{1'b0}};
    end else if (rd_en && !empty) begin  // 使能且不空
        rdata <= mem[rd_addr];            // 读出数据
    end
end
```

**特点**：
- 异步复位
- 读数据有 **1 周期延迟**（registered output）
- 空时自动忽略读取

### 4. 指针更新（核心逻辑）

```verilog
always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
        wr_ptr <= 0;
        rd_ptr <= 0;
        count  <= 0;
    end else begin
        case ({wr_en && !full, rd_en && !empty})
            2'b01: begin // 只读
                rd_ptr <= rd_ptr + 1;
                count  <= count - 1;
            end
            2'b10: begin // 只写
                wr_ptr <= wr_ptr + 1;
                count  <= count + 1;
            end
            2'b11: begin // 同时读写
                wr_ptr <= wr_ptr + 1;
                rd_ptr <= rd_ptr + 1;
                // count 不变！
            end
            default: ;   // 空闲
        endcase
    end
end
```

**关键设计**：同时读写时 `count` 不变，保持流水线平衡。

### 5. 空满信号（组合逻辑）

```verilog
assign empty = (wr_ptr == rd_ptr);           // 完全相同

assign full  = (wr_ptr[PTR_W] != rd_ptr[PTR_W]) &&      // 绕回位不同
               (wr_ptr[PTR_W-1:0] == rd_ptr[PTR_W-1:0]); // 地址相同
```

**特点**：FIFO 保持一定数据量，适合流式处理。

## 与 Gray 码 FIFO 的对比

| 特性 | 二进制指针（本设计） | Gray 码指针 |
|------|-------------------|------------|
| 空满判断 | 需要额外位，组合逻辑复杂 | 直接比较，简单 |
| 跨时钟域 | ❌ 不适合 | ✅ 适合（异步 FIFO） |
| 单时钟域性能 | ✅ 更快，指针直接加减 | 需要 Gray 码转换 |
| 面积 | 较小 | 较大（有转换逻辑） |
| 应用场景 | 同步 FIFO（单时钟） | 异步 FIFO（跨时钟域） |

## 关键设计要点

```
┌─────────────────────────────────────────┐
│  1. 指针比地址多 1 位（MSB 为绕回标志）   │
│  2. 满判断：地址相同，但绕回位不同        │
│  3. 空判断：指针完全相同                 │
│  4. 同时读写时 count 不变（流水线平衡）   │
│  5. 读写都有 1 周期延迟（时序优化）       │
└─────────────────────────────────────────┘
```

这个设计是 GPU/AI 加速器中数据缓冲、任务队列的标准实现方式。
